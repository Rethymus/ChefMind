# 贡献指南

感谢您对 ChefMind 智食谱项目的关注！我们欢迎任何形式的贡献，包括但不限于：

- 🐛 报告 Bug
- 💡 提出新功能建议
- 📝 改进文档
- 🔧 提交代码修复
- ✨ 添加新功能

## 🚀 快速开始

### 1. Fork 项目

点击项目页面右上角的 "Fork" 按钮，将项目 fork 到您的 GitHub 账户。

### 2. 克隆项目

```bash
git clone https://github.com/your-username/chefmind-vue.git
cd chefmind-vue
```

### 3. 安装依赖

```bash
npm install
```

### 4. 启动开发服务器

```bash
npm run dev
```

## 📋 开发规范

### 代码风格

项目使用 ESLint 和 Prettier 进行代码规范检查：

```bash
# 检查代码规范
npm run lint

# 自动修复代码格式
npm run format

# TypeScript 类型检查
npm run type-check
```

### 提交规范

我们使用 [Conventional Commits](https://www.conventionalcommits.org/zh-hans/) 规范：

```
<类型>[可选的作用域]: <描述>

[可选的正文]

[可选的脚注]
```

#### 类型说明

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整（不影响代码运行）
- `refactor`: 代码重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动
- `ci`: CI/CD 相关

#### 示例

```bash
git commit -m "feat: 添加食材搜索功能"
git commit -m "fix: 修复菜谱生成时的空指针异常"
git commit -m "docs: 更新 README 中的安装说明"
```

### 分支命名规范

- `feature/功能名称` - 新功能开发
- `fix/问题描述` - Bug 修复
- `docs/文档更新` - 文档更新
- `refactor/重构内容` - 代码重构

## 🔧 开发流程

### 1. 创建分支

```bash
# 从 main 分支创建新分支
git checkout -b feature/your-feature-name
```

### 2. 开发功能

- 编写代码
- 添加测试（如果适用）
- 更新文档

### 3. 提交代码

```bash
# 添加文件到暂存区
git add .

# 提交代码（遵循提交规范）
git commit -m "feat: 添加新功能描述"
```

### 4. 推送分支

```bash
git push origin feature/your-feature-name
```

### 5. 创建 Pull Request

1. 访问您 fork 的仓库页面
2. 点击 "New Pull Request"
3. 选择要合并的分支
4. 填写 PR 标题和描述
5. 提交 Pull Request

## 📝 Pull Request 指南

### PR 标题

使用清晰、简洁的标题描述您的更改：

- ✅ `feat: 添加食材分类筛选功能`
- ✅ `fix: 修复移动端布局问题`
- ❌ `更新代码`
- ❌ `修复 bug`

### PR 描述

请在 PR 描述中包含以下信息：

```markdown
## 📋 变更类型
- [ ] Bug 修复
- [ ] 新功能
- [ ] 文档更新
- [ ] 代码重构
- [ ] 性能优化

## 📝 变更描述
简要描述您的更改内容...

## 🧪 测试
- [ ] 已在本地测试
- [ ] 已添加单元测试
- [ ] 已测试响应式布局

## 📷 截图（如果适用）
如果是 UI 相关的更改，请提供截图

## 🔗 相关 Issue
关闭 #issue_number
```

### 代码审查

所有 PR 都需要经过代码审查才能合并。审查者会检查：

- 代码质量和风格
- 功能实现的正确性
- 测试覆盖率
- 文档完整性
- 性能影响

## 🐛 报告 Bug

### 使用 Issue 模板

请使用我们提供的 Bug 报告模板，包含以下信息：

- **Bug 描述**: 清晰、简洁地描述 bug
- **复现步骤**: 详细的复现步骤
- **预期行为**: 描述您期望发生的情况
- **实际行为**: 描述实际发生的情况
- **环境信息**: 浏览器、操作系统、Node.js 版本等
- **截图**: 如果适用，请提供截图

### Bug 报告示例

```markdown
**Bug 描述**
在移动端设备上，食材选择页面的搜索框无法正常输入

**复现步骤**
1. 使用手机浏览器访问应用
2. 进入食材选择页面
3. 点击搜索框
4. 尝试输入文字

**预期行为**
应该能够正常输入搜索关键词

**实际行为**
搜索框无法获得焦点，无法输入文字

**环境信息**
- 设备: iPhone 12
- 浏览器: Safari 15.0
- 操作系统: iOS 15.0
```

## 💡 功能建议

我们欢迎新功能建议！请在提交建议前：

1. 检查是否已有类似的建议
2. 确保建议符合项目目标
3. 提供详细的功能描述和使用场景

### 功能建议模板

```markdown
**功能描述**
简要描述您建议的功能

**使用场景**
描述这个功能的使用场景和用户价值

**详细设计**
如果有具体的设计想法，请详细描述

**替代方案**
是否考虑过其他实现方式？

**附加信息**
任何其他相关信息或参考资料
```

文档改进包括：

- 修复错别字和语法错误
- 改进说明的清晰度
- 添加示例和教程
- 翻译文档

## 🧪 测试

### 运行测试

```bash
# 运行所有测试
npm run test

# 运行测试并生成覆盖率报告
npm run test:coverage

# 监听模式运行测试
npm run test:watch
```

### 编写测试

- 为新功能编写单元测试
- 确保测试覆盖率不降低
- 使用描述性的测试名称
- 遵循 AAA 模式（Arrange, Act, Assert）
